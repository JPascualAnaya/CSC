Browse HCNEs in external genome browsers

The Ancora genome browser provides the most flexible way to explore the Ancora data. However, it is also useful to view the data in other browsers where it can be compared to annotations not present on this site.

UCSC Genome Browser

All files in our downloads section can be uploaded as custom tracks to the UCSC Genome Browser. Please visit the downloads section for more details.

Ensembl

Much of the Ancora data is accessible through the DAS protocol for exchanging sequence annotations. Ensembl can display sequence annotations provided in this way. Enabling Ancora data in Ensembl is easy:

  1. Go to the Ensembl web site and bring up a genome browser for human, mouse, zebrafish or fruitfly. For example: click "Human" on Ensembl’s start page, and then click "Location" in the menu to the left.
  2. Click "Add cutom data to page" in the menu to the left.
  3. Click "Attach DAS". Enter http://ancora.genereg.net/das in the field labeled "other DAS server" and click "Next".
  4. Select the tracks you wish to add. Note that the list only shows tracks that are valid for the genome that you are currently looking at in Ensembl. Click "Next".
  5. Click "Close" in the upper right corner. The Ancora tracks that you have selected should now be visible in the lower panel (the one that shows a detailed view of the selected region). If you do not see the Ancora tracks, the reason may be that there are no HCNEs in the region that you are looking at. In that case, navigate to a different region (for example, that of your favourite developmental regulator).

To remove any tracks that you have added, click "Add custom data to page" again and select "Manage data".

Note that the Ancora data can be viewed on both the "Region in detail" and the "Region overview" view in Ensembl. The latter is particularly useful for inspecting HCNE density distributions over large genomic regions.

Some users may recall that we used to have a direct link here to a form for adding data to ContigView in Ensembl. We have disabled this form as it made use of a feature that is not yet supported by Ensembl's new web interface.
